## ⚠️ 배포 전 반드시 해결해야 할 것

### 1. Redis (리프레시 토큰 저장소)
> Cloud Run에는 Redis가 없어서 외부 서비스가 필요함.
> **Upstash** 무료 플랜 사용 권장 (신용카드 불필요)

1. [https://upstash.com](https://upstash.com) 접속 → Google 계정으로 가입
2. `Create Database` → 이름: `comma-redis` → Region: `ap-northeast-1 (Tokyo)` 선택
3. 생성 후 **Endpoint(Host)** 와 **Password** 메모해두기

## 📋 배포 전 준비 체크리스트

### GCP 관련
- [ ] Google Cloud Console 로그인 확인 → [https://console.cloud.google.com](https://console.cloud.google.com)
- [ ] 프로젝트 ID 확인해두기 (예: `comma-project-123`)
- [ ] Cloud SQL 인스턴스 연결 이름 확인
  - Cloud SQL 콘솔 → 인스턴스 클릭 → `연결 이름` 복사
  - 형식: `프로젝트ID:리전:인스턴스명` (예: `comma-project:asia-northeast3:comma-db`)
- [ ] `gcloud` CLI 설치 확인
  ```bash
  gcloud --version
  ```
  - 없으면 설치: [https://cloud.google.com/sdk/docs/install](https://cloud.google.com/sdk/docs/install)
- [ ] gcloud 로그인
  ```bash
  gcloud auth login
  gcloud config set project [프로젝트ID]
  ```

### Redis 관련
- [ ] Upstash 가입 → Redis 인스턴스 생성
- [ ] Host(Endpoint)와 Password 메모

### Gmail 앱 비밀번호 (이메일 발송용)
- [ ] Google 계정 → 보안 → 2단계 인증 활성화
- [ ] `앱 비밀번호` 생성 → 메모

## 🔑 환경변수 목록 (Cloud Run 배포 시 입력)

아래 값들을 미리 메모장에 정리해두고 배포할 때 붙여넣기

```
INSTANCE_CONNECTION_NAME = 프로젝트ID:리전:인스턴스명
DB_NAME                  = comma_db
DB_USERNAME              = commatest
DB_PASSWORD              = Comma_1234
JWT_SECRET               = comma-jwt-secret-key-2026-change-this
REDIS_HOST               = (Upstash Endpoint)
REDIS_PASSWORD           = (Upstash Password)
MAIL_USERNAME            = (Gmail 주소)
MAIL_PASSWORD            = (Gmail 앱 비밀번호)
APP_BASE_URL             = (백엔드 배포 후 발급된 Cloud Run URL)
APP_FRONT_URL            = (프론트 배포 후 발급된 Cloud Run URL)
CLAUDE_API_KEY           = (선택사항)
```

> ⚠️ `APP_BASE_URL`과 `APP_FRONT_URL`은 배포 후 URL이 나오면 다시 업데이트 필요

## 🚀 배포 순서

### STEP 1. 백엔드 Cloud Run 배포

```bash
cd /Users/joymin/Desktop/파이널/commatest99/backend

gcloud run deploy comma-backend \
  --source . \
  --region asia-northeast3 \
  --platform managed \
  --allow-unauthenticated \
  --port 8080 \
  --set-env-vars INSTANCE_CONNECTION_NAME=프로젝트ID:리전:인스턴스명 \
  --set-env-vars DB_NAME=comma_db \
  --set-env-vars DB_USERNAME=commatest \
  --set-env-vars DB_PASSWORD=Comma_1234 \
  --set-env-vars JWT_SECRET=comma-jwt-secret-key-2026 \
  --set-env-vars REDIS_HOST=Upstash_Host \
  --set-env-vars REDIS_PASSWORD=Upstash_Password \
  --set-env-vars MAIL_USERNAME=gmail주소 \
  --set-env-vars MAIL_PASSWORD=앱비밀번호
```

완료되면 `https://comma-backend-xxxx-an.a.run.app` 형태의 URL이 발급됨 → 메모

---

### STEP 2. 프론트엔드 배포 전 API URL 수정

```bash
# frontend/.env.production 파일 확인/수정
REACT_APP_API_URL=https://comma-backend-xxxx-an.a.run.app
```

또는 `frontend/nginx.conf`에서 proxy_pass 백엔드 URL 확인

---

### STEP 3. 프론트엔드 Cloud Run 배포

```bash
cd /Users/joymin/Desktop/파이널/commatest99/frontend

gcloud run deploy comma-frontend \
  --source . \
  --region asia-northeast3 \
  --platform managed \
  --allow-unauthenticated \
  --port 80
```

완료되면 `https://comma-frontend-xxxx-an.a.run.app` 형태의 URL이 발급됨

---

### STEP 4. 백엔드 환경변수 업데이트 (프론트 URL 추가)

```bash
gcloud run services update comma-backend \
  --region asia-northeast3 \
  --set-env-vars APP_BASE_URL=https://comma-backend-xxxx-an.a.run.app \
  --set-env-vars APP_FRONT_URL=https://comma-frontend-xxxx-an.a.run.app
```

---

## 🆘 문제 상황별 해결법

| 증상 | 원인 | 해결 |
|------|------|------|
| 로그인이 안 됨 | Redis 연결 실패 | Upstash Host/Password 확인 |
| API 호출이 안 됨 | CORS 오류 | 백엔드 APP_FRONT_URL 환경변수 확인 |
| DB 연결 실패 | Cloud SQL 연결 이름 오류 | INSTANCE_CONNECTION_NAME 확인 |
| 빌드 실패 | Gradle/Node 오류 | Cloud Build 로그 확인 |
