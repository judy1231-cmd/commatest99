# [PART 1] 오프닝 & 프로젝트 소개 (약 1분 30초)

(슬라이드: 타이틀 화면)

안녕하세요 저는 이번에 개인으로 프로젝트 하게 된 강민정입니다.
먼저 말씀드리고 싶은 건 저는 이런 발표가 처음이라서 실수 할 수 있고 헛점이 많을 수 있는 점 양해 부탁드립니다.
오늘 제가 발표할 프로젝트는 '쉼표(,)' — 당신에게 필요한 휴식을 찾아드립니다 라는
개인 맞춤형 휴식 추천 플랫폼입니다.

프로젝트 이름을 쉼표로 정한 데에는 이유가 있습니다.
문장에서 쉼표는 "잠깐 멈추고, 다음으로 넘어가는 기호"입니다.
바쁜 일상 속에서 잠깐 멈추고 나에게 맞는 휴식을 취할 수 있도록 돕는 서비스입니다.

오늘 발표 순서는 주제선정배경 → 벤치마킹 → 주요기능, 린캔버스 → 유스케이스 →
→ 실제 구현 데모 → 향후 계획 순으로 진행하겠습니다.

# [PART 2] 주제 선정 배경 (약 2분 30초)

(슬라이드: 주제선정배경 — 번아웃/시간부족/방법모름)

먼저 왜 이 주제를 선택했는지 말씀드리겠습니다.

현대인의 70% 이상이 번아웃을 경험한다는 통계가 있습니다.
다들 피로하다는 건 알고 있습니다. 그런데 막상 "쉬어야지"라고 생각하면서도 세
가지 문제에 막힙니다.

첫째, 시간이 없습니다. 직장인은 퇴근 후 몇 시간, 학생은 쉬는 시간 몇 분이 전부입니다.
둘째, 방법을 모릅니다. 쉬고 싶은데 뭘 해야 할지 모르겠다는 사람이 생각보다 훨씬 많습니다.
셋째, 기존 앱들은 추천이 없거나, 너무 비싸거나, 내 상태를 고려하지 않습니다.

(슬라이드: "쉬고 싶은데 어떻게 쉬어야 할지 모르겠어요")

이 슬라이드가 핵심입니다.
사용자에게 필요한 건 막연한 자유 시간이 아닙니다.
"지금 내 몸과 마음 상태에 맞는 휴식 처방" 이 필요합니다.

(슬라이드: 쉼표 개념 — 막연한 휴식을 구체적 '경험'으로)

그래서 쉼표는 세 가지를 지향합니다.
- 심리 기반 추천: 심리 테스트 + 심박수 분석으로 내 상태를 진단하고
- 자연 테마: 일상 속에서 접근 가능한 자연 기반 장소를 연결하며
- 즉시 실천 가능: 생각만 하는 게 아니라 바로 실행할 수 있는 가이드를 제공합니다.

# [PART 3] 벤치마킹 (약 2분 30초)

(슬라이드: 벤치마킹 — 무디앱)

기획 단계에서 세 가지 서비스를 벤치마킹했습니다.

첫 번째는 무디앱입니다.
감성적인 UI와 기분 트래킹은 훌륭하지만, 단순 기록에 그칩니다.
"오늘 기분이 어때요?"는 물어보는데, "그럼 이렇게 해봐요"는 없습니다.
→ 쉼표는 여기서 '추천'과 '행동 가이드'를 추가합니다.

(슬라이드: 에어비앤비)

두 번째는 에어비앤비입니다.
카테고리별 큐레이션과 지역 기반 추천은 탁월합니다.
하지만 숙박/여행 중심이라 일상적인 휴식엔 맞지 않고, 금전적 부담도 큽니다.
→ 쉼표는 일상 속에서 무료 또는 저비용으로 접근 가능한 장소에 집중합니다.

(슬라이드: 핀터레스트)

세 번째는 핀터레스트입니다.
AI 기반 개인화 추천과 비주얼 영감 제공은 좋지만, 실행으로 이어지지 않습니다.
보다가 오히려 더 피로해지는 경험이 생깁니다.
→ 쉼표는 '보는 것'에서 '실행하는 것'으로 연결하는 단계를 핵심으로 설계했습니다.

(슬라이드: 주요기능 6가지)

이 세 서비스의 장점을 흡수하고 단점을 보완해서 쉼표의 6가지 핵심 기능을 도출했습니다:
사용자 선택형 휴식, 심리테스트 추천, 심박수 기반 추천, 산책 안내, 명상 가이드, 특별 체험입니다.

# [PART 4] 린캔버스 (약 1분 30초)

(슬라이드: 린캔버스)

린캔버스로 비즈니스 모델을 정리했습니다.

문제: 번아웃 경험, 휴식 방법 부재, 기존 서비스의 한계
고객군: 직장인, 수험생, 육아 중인 부모, 크리에이터 — 스트레스 고위험군 전반입니다.
솔루션: 심박수와 심리 설문을 결합해 컨디션을 분석하고, 단계별 가이드를 제공합니다.
가치 제안: 단순 정보 제공이 아니라 "즉시 실행 가능한 휴식 처방" 입니다.
경쟁우위: 카메라 센서 기반 실시간 심박수 분석과 실행 중심 콘텐츠 라이브러리가 차별점입니다.
핵심 지표: 주간 활성 사용자 수, 추천 가이드 완료율, 서비스 이용 전후 스트레스 지수 변화입니다.

# [PART 6] 유스케이스 (약 1분)

(슬라이드: 유스케이스 다이어그램)

유스케이스 다이어그램으로 사용자와 관리자의 핵심 흐름을 정의했습니다.

사용자는 회원가입 및 로그인 또는 심리상태 진단을 통해 서비스에 진입하고,
→ 맞춤형 휴식 추천 → 사용자 선택형 휴식 → 산책 가이드, 명상 프로그램, 자연테마 콘텐츠 감상으로 이어지며
→ 마지막으로 휴식 기록 확인과 통계 분석으로 개선 사이클이 완성됩니다.

핵심 흐름은 진단 → 추천 → 활동 → 기록 → 개선 이 다섯 단계입니다.

# [PART 8] 실제 구현 데모 (약 5분)

## 1단계. 회원가입 & 로그인 (약 40초)

[행동] 주소창에 localhost:3000/signup 입력 또는 메인에서 회원가입 클릭

회원가입 화면입니다. 이메일, 비밀번호, 아이디를 입력하면 가입이 완료됩니다.

[행동] 이미 가입된 테스트 계정으로 바로 로그인 페이지로 이동 localhost:3000/login

[행동] 이메일: test@comma.com / 비밀번호: Test@1234 입력 → 로그인 버튼 클릭

로그인 성공 시 JWT 토큰이 발급되고 메인 대시보드로 이동합니다.
서버에서 발급된 토큰은 로컬스토리지에 저장되어 이후 모든 API 요청에 사용됩니다.

## 2단계. 메인 대시보드 (약 30초)

[행동] 로그인 후 메인 페이지 (localhost:3000/) 확인

메인 화면입니다. 상단에 심박수 체크와 휴식 유형 테스트 진입 카드가 있고,
가운데에는 휴식 카테고리 7종 탭이 있습니다.

[행동] 카테고리 탭 중 '자연의 연결', '정신적 휴식' 등 클릭해서 탭 전환 시연

[행동] 스크롤 내려서 하단 월간 통계 영역 보여주기

## 3단계. 심박수 측정 (약 50초)

[행동] 상단 심박수 체크 카드 클릭 → localhost:3000/heartrate 이동

심박수 측정 화면입니다.
이 QR코드를 스마트폰으로 찍거나, URL을 복사해서 모바일 브라우저에서 접속하면
카메라 센서로 심박수를 측정해 이 화면으로 데이터가 전송됩니다.

[행동] "측정 시작" 버튼 클릭 → 측정 중 애니메이션 확인

## 4단계. 휴식 유형 테스트 (약 1분)

[행동] 메인으로 돌아가서 "휴식 유형 테스트" 카드 클릭 → localhost:3000/rest-test 이동

휴식 유형 진단 테스트입니다. 총 12문항으로 구성되어 있습니다.

[행동] 빠르게 3~4문항 연속 클릭 → 12문항 완료

12문항이 완료되면 7가지 휴식 유형 각각의 점수가 계산되어 나의 주요 휴식 유형이 도출됩니다.
유형별 점수는 DB의 rest_type_scores 테이블에 저장되고, 이후 장소 추천의 기준이 됩니다.

## 5단계. 휴식 기록 (약 1분)

[행동] 상단 네비게이션 → '기록' 클릭 → localhost:3000/rest-record 이동

[행동] 휴식 유형 선택 → 시작/종료 시간 입력 → 메모 입력 → 감정 슬라이더 조절 → 저장

휴식 전후 감정 점수를 1~10점으로 기록합니다.
이 데이터가 쌓이면 "내가 어떤 휴식을 했을 때 감정이 개선됐는지" 분석이 가능해집니다.

[행동] 저장 후 하단 월간 통계 영역 확인

## 6단계. 휴식 지도 (약 20초)

[행동] 상단 네비게이션 → '지도' 클릭 → localhost:3000/map 이동

공공데이터포털 기반으로 구성되어 있으며, 주변 휴식 장소 핀이 표시됩니다.

[행동] 좌측 카테고리 탭 클릭 → 지도 마커 하나 클릭 → 장소 카드 팝업 확인

## 7단계. 관리자 페이지 (약 30초)

[행동] 주소창에 localhost:3000/admin/login 입력

[행동] 이메일: admin@comma.com / 비밀번호: Admin@1234 입력 → 로그인

[행동] 관리자 대시보드 확인 → 좌측 사이드바 '사용자 관리' 클릭

# [PART 9] DB 설계 (약 1분)

DB는 총 27개 테이블로 설계했으며, 7개 도메인으로 분류됩니다.

핵심 데이터 흐름:
회원가입 → 심박 측정 세션 → 설문 응답 → 진단 결과 → 장소 추천 → 휴식 기록 → 월간 통계

- 추천 로그 반드시 저장 — 클릭/저장 여부 기록해야 통계가 의미 있음

# Q&A 스크립트

**Q1. 심박수는 어떻게 측정해요? 카메라로 되나요?**
A. 네, PPG(광혈류측정) 방식입니다. 스마트폰 카메라를 손가락으로 덮으면 빛의 반사량 변화로 심박수를 측정합니다. Apple Watch, Samsung Health도 동일한 원리를 사용합니다. QR코드로 모바일 브라우저에서 접속하면 바로 측정할 수 있습니다.

**Q2. 지도는 어떤 API 사용했어요?**
A. 공공데이터포털 API를 사용했습니다. 지도 렌더링은 OpenStreetMap과 Leaflet 라이브러리를 사용했습니다. 별도 비용 없이 신뢰도 높은 장소 정보를 사용할 수 있다는 장점이 있습니다.

**Q3. 카카오맵 말고 OpenStreetMap을 쓴 이유가 뭔가요?**
A. 카카오맵 API는 사용량 제한과 유료 과금 구조가 있어서 부담스러웠습니다. OpenStreetMap은 오픈소스 무료 지도고, 공공 데이터 기반 서비스라는 취지에도 더 잘 맞는다고 판단했습니다.

**Q4. 장소 데이터는 어디서 가져왔어요?**
A. 공공데이터포털에서 공원, 도서관, 산책로 등의 데이터를 Python으로 크롤링해서 DB에 저장했습니다. 현재는 서울 지역 위주로 확보했고, 이후 전국으로 확대할 계획입니다.

**Q5. 7가지 휴식 유형은 어떻게 정한 거예요?**
A. 심리학에서 실제로 사용하는 휴식 유형 분류를 참고했습니다. 신체적 이완, 정신적 고요, 감각의 정화, 정서적 지지, 사회적 휴식, 자연의 연결, 창조적 몰입 — 각각 다른 피로 원인에 대응하는 휴식 방식입니다. 설문 12문항 답변 점수를 유형별로 평균 내어 가장 높은 유형을 추천합니다.

**Q6. 혼자 다 만든 거예요?**
A. 네, 기획부터 DB 설계, 백엔드, 프론트엔드까지 전부 혼자 진행했습니다. 백엔드는 Spring Boot와 MyBatis, 프론트엔드는 React와 Tailwind CSS, DB는 MySQL, 인증은 JWT 기반으로 구현했습니다.

**Q7. 제일 어려웠던 부분이 뭐예요?**
A. DB 설계가 가장 어려웠습니다. 처음엔 단순하게 시작했는데 기능이 늘어날수록 FK가 꼬였습니다. 결국 원천 데이터와 산출물을 분리하는 방향으로 재설계했습니다.

**Q8. Spring Security는 왜 안 썼어요?**
A. 설정이 복잡하고 이 프로젝트 규모에서는 오버엔지니어링이 될 수 있다고 판단했습니다. 대신 HandlerInterceptor 기반으로 JWT 인증을 직접 구현했습니다.

**Q9. 심박수 측정이 정확한가요?**
A. 의료용 수준은 아니지만 스트레스 지수 판단에는 충분합니다. BPM과 HRV(심박변이도)를 함께 측정해서 자율신경계 상태까지 반영하도록 설계했습니다.

**Q10. 나중에 어떻게 발전시킬 거예요?**
A. 장소 데이터 전국 확대, Claude API 활용 AI 추천 고도화, 같은 휴식 유형 사람들끼리 장소를 공유하는 커뮤니티 기능을 계획하고 있습니다.

**Q11. DB가 27개 테이블인데 처음부터 다 설계한 건가요?**
A. 멘토링을 받으면서 계속 다듬었습니다. "원천 데이터와 산출물을 분리하라", "추천 로그를 반드시 저장해야 통계가 살아난다"는 피드백을 받으면서 테이블이 늘어났습니다.
