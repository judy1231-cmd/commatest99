### `global/util/ApiResponse.java`

> 모든 API 응답을 감싸는 클래스. 프론트에서 `data.success`로 성공/실패를 판별한다.

```java
@Getter                          // Lombok — 모든 필드에 getter 자동 생성 (getSuccess, getData, getMessage)
public class ApiResponse<T> {    // <T>: 제네릭. data가 User일 수도, List<Place>일 수도 있어서

    private final boolean success;    // API 성공 여부
    private final T data;             // 실제 응답 데이터 (실패 시 null)
    private final String message;     // 사용자에게 보여줄 메시지

    private ApiResponse(...) { ... }  // private: 직접 new 하지 말고 아래 팩토리 메서드 써라

    // 성공 + 데이터 + 메시지
    public static <T> ApiResponse<T> ok(T data, String message) {
        return new ApiResponse<>(true, data, message);
    }

    // 성공인데 줄 데이터가 없을 때 (예: 삭제 성공)
    public static <T> ApiResponse<T> ok(String message) {
        return new ApiResponse<>(true, null, message);
    }

    // 실패 (data는 항상 null)
    public static <T> ApiResponse<T> fail(String message) {
        return new ApiResponse<>(false, null, message);
    }
}
```

**핵심 포인트:**
- `private` 생성자 + `static` 팩토리 메서드 패턴 — `ApiResponse.ok(...)` 이렇게만 만들 수 있다.
- 프론트에서는 항상 `if (res.success) { ... } else { alert(res.message) }` 패턴으로 쓴다.

### `domain/user/model/User.java`

> DB `users` 테이블과 1:1 대응하는 클래스. MyBatis가 쿼리 결과를 이 클래스로 자동 변환한다.

```java
@Data               // getter/setter/toString/equals/hashCode 전부 자동 생성 (Lombok)
@NoArgsConstructor  // 인수 없는 기본 생성자 — MyBatis가 객체를 만들 때 이게 필요함
@AllArgsConstructor
public class User {

    private String 쉼표번호;       // PK. "쉼표0001" 형태. 절대 숫자 타입으로 바꾸지 말 것
    private String email;          // 로그인 이메일. DB에서 UNIQUE
    private String password;       // BCrypt 해시값. API 응답 전에 반드시 null로 처리
    private String username;       // 로그인 아이디. DB에서 UNIQUE
    private String nickname;       // 표시용 닉네임. 가입 시 쉼표번호와 동일. 나중에 변경 가능
    private String status;         // active / dormant / banned
    private Boolean emailVerified; // 이메일 인증 완료 여부
    private String role;           // USER / ADMIN
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

**핵심 포인트:**
- `password`는 응답 시 반드시 `user.setPassword(null)` 처리해야 한다. 그냥 내보내면 BCrypt 해시가 노출된다.
- `nickname`과 `쉼표번호`는 처음엔 같은 값이지만, 닉네임은 사용자가 바꿀 수 있고 쉼표번호는 PK라 못 바꾼다.

### `global/config/JwtUtil.java`

> JWT 토큰을 만들고, 검증하고, 안에서 값을 꺼낸다.

```java
@Component  // Spring이 Bean으로 관리. @Autowired나 생성자 주입으로 어디서든 쓸 수 있음
public class JwtUtil {

    private final Key key;                // HMAC-SHA256 서명 키. 한 번 만들면 변하지 않음
    private final long accessExpiration;  // 30분 (1800000ms)
    private final long refreshExpiration; // 14일 (1209600000ms)

    public JwtUtil(
            @Value("${jwt.secret}") String secret,                 // application.yml에서 값 주입
            @Value("${jwt.access-expiration}") long accessExpiration,
            @Value("${jwt.refresh-expiration}") long refreshExpiration
    ) {
        this.key = Keys.hmacShaKeyFor(secret.getBytes()); // 문자열 시크릿 → 암호화 키 객체로 변환
        ...
    }

    public String generateAccessToken(String 쉼표번호, String role) {
        return Jwts.builder()
                .setSubject(쉼표번호)                               // 토큰의 "주인공". 나중에 꺼내서 누구 요청인지 확인
                .claim("role", role)                               // 커스텀 데이터 추가. ADMIN/USER 구분에 쓰임
                .setIssuedAt(new Date())                           // 발급 시각
                .setExpiration(new Date(System.currentTimeMillis() + accessExpiration)) // 만료 시각
                .signWith(key, SignatureAlgorithm.HS256)           // 이 키로 서명 → 위변조 방지
                .compact();                                        // 최종 JWT 문자열 (aaa.bbb.ccc 형태)
    }

    // Refresh Token은 role 불필요 — 새 Access Token 발급에만 쓰이니까
    public String generateRefreshToken(String 쉼표번호) { ... }

    // 토큰에서 쉼표번호 꺼내기
    public String extract쉼표번호(String token) {
        return parseClaims(token).getSubject(); // setSubject()로 넣은 값을 getSubject()로 꺼냄
    }

    // 토큰이 유효한가? (서명 맞는지 + 만료 안 됐는지)
    public boolean isTokenValid(String token) {
        try {
            parseClaims(token); // 실패하면 예외 발생
            return true;
        } catch (JwtException | IllegalArgumentException e) {
            return false;       // 만료되거나 위조된 토큰
        }
    }

    // 소셜 가입 확인 전 임시 토큰 (10분짜리)
    // subject: 소셜 제공자 ID, claims에 provider/nickname/pending 담음
    public String generatePendingToken(String provider, String providerId, String nickname, String email) {
        return Jwts.builder()
                .setSubject(providerId)
                .claim("provider", provider)   // kakao / google / naver
                .claim("pending", true)        // 이 플래그가 있어야 pending 토큰임. 정식 토큰과 구분
                .setExpiration(new Date(System.currentTimeMillis() + 10 * 60 * 1000)) // 10분
                ...
    }

    private Claims parseClaims(String token) {
        return Jwts.parserBuilder()
                .setSigningKey(key)       // 서명 검증에 쓸 키
                .build()
                .parseClaimsJws(token)    // 파싱 + 서명 검증 동시에. 실패 시 예외
                .getBody();               // 페이로드(클레임) 반환
    }
}
```

### `global/config/JwtInterceptor.java`

> 모든 `/api/**` 요청을 가로채서 JWT를 확인한다. Spring Security 없이 인증을 구현한 핵심 파일.

```java
@Component
public class JwtInterceptor implements HandlerInterceptor { // HandlerInterceptor: 컨트롤러 실행 "전"에 끼어듦

    // 패턴 매칭용 — /api/admin/** 처럼 와일드카드 패턴 지원
    private static final AntPathMatcher pathMatcher = new AntPathMatcher();

    // 이 경로들은 토큰 없이도 접근 가능
    private static final String[] PUBLIC_PATHS = {
        "/api/auth/signup",   // 회원가입
        "/api/auth/login",    // 로그인
        "/api/auth/refresh",  // 토큰 갱신
        "/api/places",        // 장소 목록 (비로그인도 탐색 가능)
        "/api/challenges",    // 챌린지 목록
        // ...
    };

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {

        // CORS preflight 요청은 무조건 통과 (브라우저가 실제 요청 전에 미리 보내는 OPTIONS 요청)
        if ("OPTIONS".equalsIgnoreCase(request.getMethod())) return true;

        String path = request.getServletPath(); // 예: /api/auth/login

        // 공개 경로인지 확인
        boolean isPublic = false;
        for (String pattern : PUBLIC_PATHS) {
            if (pathMatcher.match(pattern, path)) { isPublic = true; break; }
        }

        if (isPublic) {
            // 공개 경로여도 토큰이 있으면 사용자 정보를 꺼내둠
            // → 장소 상세 페이지에서 "내가 북마크한 곳인가" 체크에 쓰임 (선택적 인증)
            String authHeader = request.getHeader("Authorization");
            if (authHeader != null && authHeader.startsWith("Bearer ")) {
                String token = authHeader.substring(7); // "Bearer " 7글자 제거
                if (jwtUtil.isTokenValid(token)) {
                    request.setAttribute("쉼표번호", jwtUtil.extract쉼표번호(token));
                    request.setAttribute("role", jwtUtil.extractRole(token));
                }
            }
            return true; // 공개 경로 → 통과
        }

        // 여기서부터는 로그인 필요한 경로
        String header = request.getHeader("Authorization");
        if (header == null || !header.startsWith("Bearer ")) {
            sendError(response, 401, "로그인이 필요합니다.");
            return false; // false를 반환하면 컨트롤러 실행 안 됨
        }

        String token = header.substring(7);
        if (!jwtUtil.isTokenValid(token)) {
            sendError(response, 401, "유효하지 않은 토큰입니다. 다시 로그인해주세요.");
            return false;
        }

        String 쉼표번호 = jwtUtil.extract쉼표번호(token);
        String role = jwtUtil.extractRole(token);

        // /api/admin/** 경로는 ADMIN만 허용
        if (pathMatcher.match("/api/admin/**", path) && !"ADMIN".equals(role)) {
            sendError(response, 403, "관리자 권한이 필요합니다.");
            return false;
        }

        // 검증 완료 → 컨트롤러에서 꺼내 쓸 수 있도록 request에 저장
        request.setAttribute("쉼표번호", 쉼표번호);
        request.setAttribute("role", role);
        return true; // 통과. 컨트롤러 실행
    }

    // JSON 형식으로 에러 응답 직접 작성
    private void sendError(HttpServletResponse response, int status, String message) throws IOException {
        response.setStatus(status);
        response.setContentType("application/json;charset=UTF-8");
        response.getWriter().write(
            String.format("{\"success\":false,\"data\":null,\"message\":\"%s\"}", message)
        );
    }
}
```

**핵심 포인트:**
- `return true` = 통과, `return false` = 여기서 응답 끝냄. 컨트롤러 실행 안 됨.
- `request.setAttribute("쉼표번호", 쉼표번호)` → 컨트롤러에서 `(String) request.getAttribute("쉼표번호")`로 꺼냄.
- 공개 경로에서도 토큰이 있으면 사용자 정보를 꺼내두는 것이 "선택적 인증"이다.

                .allowCredentials(true)              // 쿠키 포함 요청 허용

    }
}

### `global/exception/GlobalExceptionHandler.java`

> 전역 예외 처리. Controller에서 예외가 터지면 여기서 잡아서 JSON으로 응답.

```java
@RestControllerAdvice  // 모든 @RestController에 공통 적용되는 예외 핸들러
public class GlobalExceptionHandler {

    // Service에서 throw new IllegalArgumentException("이메일 중복") 하면 여기서 잡힘
    @ExceptionHandler(IllegalArgumentException.class)
    public ResponseEntity<ApiResponse<?>> handleIllegalArg(IllegalArgumentException e) {
        return ResponseEntity.badRequest().body(ApiResponse.fail(e.getMessage()));
    }

    // 그 외 모든 예외 (DB 오류, NullPointer 등)
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ApiResponse<?>> handleException(Exception e) {
        return ResponseEntity.status(500).body(ApiResponse.fail("서버 오류가 발생했습니다."));
    }
}
```

**핵심 포인트:**
- 이게 없으면 예외 발생 시 Spring이 HTML 에러 페이지를 반환한다. 프론트에서 JSON을 기대하는데 HTML이 오면 파싱 오류가 난다.
- Service에서 `throw new IllegalArgumentException("에러 메시지")`만 하면 GlobalExceptionHandler가 알아서 잡아서 `{ success: false, message: "에러 메시지" }` 형태로 응답한다.

### `domain/auth/service/AuthService.java` — 회원가입

```java
@Transactional  // 중간에 실패하면 전부 롤백. insertUser → insertEmailVerification 둘 다 취소
public User signup(String email, String username, String password) {

    // 중복 체크
    if (authMapper.countByEmail(email.trim()) > 0)
        throw new IllegalArgumentException("이미 사용 중인 이메일입니다.");
    if (authMapper.countByUsername(username.trim()) > 0)
        throw new IllegalArgumentException("이미 사용 중인 아이디입니다.");

    String 쉼표번호 = generate쉼표번호(); // DB MAX(쉼표번호) + 1 계산. "쉼표0001" → "쉼표0002"

    User user = new User();
    user.set쉼표번호(쉼표번호);
    user.setEmail(email.trim());
    user.setPassword(BCrypt.hashpw(password, BCrypt.gensalt())); // 단방향 해싱. 원래 비번 복구 불가
    user.setNickname(쉼표번호);   // 처음엔 쉼표번호 = 닉네임. 나중에 닉네임만 변경 가능
    user.setStatus("active");
    user.setEmailVerified(false);
    user.setRole("USER");

    authMapper.insertUser(user);

    // 개발 환경(app.skip-email-verification=true)이면 인증 메일 안 보내고 바로 완료 처리
    if (skipEmailVerification) {
        authMapper.updateUserEmailVerified(쉼표번호);
    } else {
        try {
            String token = UUID.randomUUID().toString(); // UUID를 인증 링크 토큰으로 사용
            authMapper.insertEmailVerification(쉼표번호, token, LocalDateTime.now().plusHours(24));
            mailService.sendVerificationEmail(email.trim(), username.trim(), token);
        } catch (Exception e) {
            // 메일 발송 실패해도 가입 자체는 성공 처리 (사용자가 이미 DB에 생성됨)
            log.error("[메일 발송 실패] to={}", email.trim(), e);
        }
    }

    return user;
}
```

### `domain/auth/service/AuthService.java` — 로그인

```java
public Map<String, Object> login(String email, String password) {

    User user = authMapper.findByEmail(email.trim());
    if (user == null)
        throw new IllegalArgumentException("이메일 또는 비밀번호가 올바르지 않습니다.");
        // ↑ "이메일이 없습니다"가 아닌 같은 메시지 → 공격자에게 이메일 존재 여부를 알려주면 안 됨

    // BCrypt.checkpw: 입력 비번과 DB 해시값 비교
    if (!BCrypt.checkpw(password, user.getPassword()))
        throw new IllegalArgumentException("이메일 또는 비밀번호가 올바르지 않습니다.");

    if ("banned".equals(user.getStatus()))
        throw new IllegalArgumentException("이용이 제한된 계정입니다.");

    String accessToken = jwtUtil.generateAccessToken(user.get쉼표번호(), user.getRole());
    String refreshToken = jwtUtil.generateRefreshToken(user.get쉼표번호());

    // Refresh Token을 Redis에 저장 (Key: "RT:쉼표번호", TTL: 14일)
    redisTemplate.opsForValue().set(
        REFRESH_TOKEN_PREFIX + user.get쉼표번호(),
        refreshToken,
        refreshExpiration, TimeUnit.MILLISECONDS
    );

    user.setPassword(null); // 응답에 비밀번호 포함되면 안 됨 → null로 지움

    return Map.of(
        "accessToken", accessToken,
        "refreshToken", refreshToken,
        "user", user
    );
}
```

### `api/fetchWithAuth.js`

> JWT가 필요한 API 호출 wrapper. 토큰 만료 시 자동으로 갱신하고 재시도한다.

```javascript
let isRefreshing = false;  // 지금 토큰 갱신 중인가? (전역 플래그)
let refreshQueue = [];     // 갱신 중에 들어온 요청들을 여기 대기시킴

// 갱신 완료되면 대기 중인 요청들에게 새 토큰 전달해서 재시도
function processRefreshQueue(newToken) {
  refreshQueue.forEach(({ resolve }) => resolve(newToken));
  refreshQueue = [];
}

// 강제 로그아웃: localStorage 비우고 /login으로 이동
function forceLogout() {
  localStorage.removeItem('accessToken');
  localStorage.removeItem('refreshToken');
  localStorage.removeItem('user');
  window.location.href = '/login';
}

export async function fetchWithAuth(url, options = {}) {
  const accessToken = localStorage.getItem('accessToken');

  const headers = {
    'Content-Type': 'application/json',  // 기본값. FormData 쓸 때는 이걸 지워야 boundary가 자동 설정됨
    ...options.headers,                   // 호출 측에서 덮어쓸 수 있음
  };

  if (accessToken) {
    headers['Authorization'] = `Bearer ${accessToken}`;
  }

  let res = await fetch(url, { ...options, headers });

  // 401 = 토큰 만료
  if (res.status === 401 && localStorage.getItem('refreshToken')) {
    try {
      let newToken;

      if (isRefreshing) {
        // 이미 갱신 중이면 큐에 등록하고 대기
        // 여러 요청이 동시에 401을 받아도 갱신은 1번만 실행됨
        newToken = await new Promise((resolve, reject) => {
          refreshQueue.push({ resolve, reject });
        });
      } else {
        isRefreshing = true;
        try {
          newToken = await refreshAccessToken(); // Refresh Token으로 새 Access Token 발급
          processRefreshQueue(newToken);          // 대기 중인 요청들도 재시도
        } catch (refreshError) {
          rejectRefreshQueue(refreshError);
          forceLogout();
          throw refreshError;
        } finally {
          isRefreshing = false;
        }
      }

      // 새 토큰으로 원래 요청 재시도
      headers['Authorization'] = `Bearer ${newToken}`;
      res = await fetch(url, { ...options, headers });
    } catch (e) {
      forceLogout();
      throw e;
    }
  }

  return res.json(); // 최종 응답을 JSON으로 파싱해서 반환
}
```

**주의사항:**
- 파일 업로드(FormData)할 때는 `fetchWithAuth` 쓰면 안 된다. `Content-Type: application/json`이 강제 설정돼서 multipart boundary가 날아간다.
- 파일 업로드는 raw `fetch()`를 직접 쓰고 Authorization 헤더만 수동으로 추가한다:

```javascript
// 파일 업로드 시 이렇게
const res = await fetch('/api/upload', {
  method: 'POST',
  headers: { Authorization: `Bearer ${localStorage.getItem('accessToken')}` },
  // Content-Type 없음 → 브라우저가 multipart/form-data; boundary=... 자동 설정
  body: formData,
});
```

@Service
public class AnalyticsService {

    @Async  // 이 어노테이션 하나로 비동기 처리. 호출한 쪽은 이 메서드가 끝날 때까지 기다리지 않음
    public void track(String commaNo, String eventType, Map<String, Object> data) {
        try {
            String json = data == null ? null : toJson(data);
            adminMapper.insertAnalyticsEvent(commaNo, eventType, json);
        } catch (Exception e) {
            // 실패해도 경고 로그만 남기고 무시. 이벤트 수집 실패가 메인 기능을 막으면 안 됨
            log.warn("analytics track 실패: {} - {}", eventType, e.getMessage());
        }
    }

    // Jackson 없이 직접 JSON 직렬화 — 외부 의존성 추가 안 하려고
    private String toJson(Map<String, Object> data) {
        StringBuilder sb = new StringBuilder("{");
        boolean first = true;
        for (Map.Entry<String, Object> entry : data.entrySet()) {
            if (!first) sb.append(",");
            sb.append("\"").append(entry.getKey()).append("\":");
            Object val = entry.getValue();
            if (val == null)                                    sb.append("null");
            else if (val instanceof Number || val instanceof Boolean) sb.append(val); // 따옴표 없이
            else sb.append("\"").append(val.toString().replace("\"", "\\\"")).append("\""); // 이스케이프
            first = false;
        }
        return sb.append("}").toString();
    }
}
